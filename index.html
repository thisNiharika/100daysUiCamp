Week 1 - Days 1–10 - Focus: build, test, iterate

Day 1 - Glassmorphic Login Card

Tasks

Design a centered glassmorphic login card: blurred translucent background (backdrop-filter), clear form fields with accessible labels, large visible focus states, a primary CTA, 'Forgot password' link, and a small sign-up secondary action. Include responsive spacing so it works on mobile and desktop; ensure sufficient contrast for text over blur and provide a non-blurred fallback.

Day 1

Day 2 - 3D Hover Card Tilt

Tasks

Create a product/info card that tilts using a 3D transform on pointer hover and returns smoothly on pointer out. Include subtle shadow, preserved 3D perspective, reduced-motion fallback, and keyboard activation (focus triggers same effect). The card should contain image, title, short description, and an actionable button.

Day 2

Day 3 - Animated Gradient Text

Tasks

Build a heading component that uses an animated linear-gradient moving left-to-right clipped to text (background-clip:text). Provide a solid-color fallback for browsers without text-clip support. Ensure text remains readable (contrast and accessible font-size) and allow pause-on-hover for users.

Day 3

Day 4 - Neumorphic Toggle Switch

Tasks

Craft a soft 'neumorphic' toggle switch that visually depresses when toggled: use dual shadows for light/dark illusion. Make it keyboard operable and announce state with aria-checked. Provide a clear active and inactive label and consider contrast for accessibility.

Day 4

Day 5 - Floating Action Button with Ripple

Tasks

Implement a floating action button (FAB) that sits in the bottom-right corner, with a centered icon and an ink/ripple effect on click. Ensure keyboard activation triggers the same visual feedback, and provide an accessible label (aria-label). Include elevation (shadow) and an optional secondary mini action menu on long press/click-and-hold.

Day 5

Day 6 - Responsive Image Grid with Hover Overlay

Tasks

Create a responsive masonry-like or grid gallery of images. On hover (or focus), show a semi-transparent overlay with a caption and two actions (view & like). Provide keyboard focus states that reveal overlays and include accessible alt text and captions that are discoverable by screen readers.

Day 6

Day 7 - Animated Navigation Underline

Tasks

Design a horizontal nav where an animated underline smoothly slides to the active link. Support hover, focus, and active route state. Use transforms for performant animation and ensure the underline's size adapts to link width; provide focus-visible styles for keyboard users.

Day 7

Day 8 - Responsive Pricing Cards

Tasks

Build three pricing cards (Basic / Pro / Enterprise) with feature lists, prices, and a prominent 'Recommended' highlight for the middle plan. Cards must stack on small screens, include accessible semantics (ul for features, ARIA for current plan), and show monthly/yearly toggle behavior (UI only).

Day 8

Day 9 - Loader — Rotating Dots

Tasks

Create a minimal loader made of three rotating or bouncing dots with staggered timing. Ensure it announces 'Loading' with an accessible live region and respects prefers-reduced-motion by showing a static alternative or simpler animation.

Day 9

Day 10 - Sticky Header with Scroll Shadow

Tasks

Implement a header that becomes fixed (sticky) after scrolling past hero area and gains a subtle shadow and reduced height. Include skip-to-content link for accessibility and ensure the header does not obscure content (use appropriate top spacing).
